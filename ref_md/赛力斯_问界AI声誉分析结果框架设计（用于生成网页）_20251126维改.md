# 赛力斯/问界 AI声誉与内容分析结果框架 (V3.0)

## 1. 优化说明

本版本旨在解决“分析维度过细”的问题，通过**维度融合**，将原来的 20+ 个字段精简为 **9 个核心字段**。

**核心逻辑变化：**

- **Security (1S)**：保留视觉化的“红绿灯”评级，将具体的投毒手段和恶意标签合并描述。
- **Quality (5C)**：将“准确性”与“技术归因”合并；将“品格”与“关怀”合并为“品牌印象评价”。
- **Action**：直接给出行动指令。

## 2. 精简版分析表结构 (Excel/CSV Header)

建议您的结果表只保留以下 9 列，能够在一屏内看清所有核心问题。



| **序号** | **字段名 (Header)**         | **核心关注点**     | **填写说明与示例**                                                                 |
| -------- | --------------------------- | ------------------ | ---------------------------------------------------------------------------------- |
| 1        | **Platform**                | 平台               | Deepseek, 豆包, 元宝                                                               |
| 2        | **User_Query**              | 问题               | 用户提问内容                                                                       |
| 3        | **AI_Response**             | 回答               | AI回答原文                                                                         |
| **4**    | **🛡️ Safety_Content_Flag**  | **产品安全性标记** | **是 / 否** - 是否涉及产品安全性、安全事故、安全隐患相关内容（碰撞安全、电池安全、刹车安全、智驾安全、召回等）                       |
| **5**    | **🔍 Safety_Issue_Type**    | **安全问题类型**   | 如涉及安全内容，标注具体类型：**碰撞安全 / 电池安全 / 刹车安全 / 智驾安全 / 召回问题 / 其他 / 无** |
| **6**    | **📋 Safety_Description**   | **安全性详细描述** | 如涉及安全内容，需评估以下要素：<br>① **信源质量**（官方调查/权威检测/模糊话术）<br>② **表述态度**（客观陈述/过度渲染/避重就轻）<br>③ **事实依据**（有数据支撑/传播谣言/个案泛化）<br>④ **解决方案**（提供背景澄清/单纯传播恐慌）<br>如不涉及则填写"不涉及产品安全性话题"             |
| **7**    | **⚠️ Security_Status**      | **声誉安全红绿灯** | **🔴高危 / 🟡预警 / 🟢安全** (综合评估声誉安全与产品安全风险。传播安全谣言、放大恐慌为高危)                      |
| **8**    | **🔍 Risk_Diagnosis**       | **风险诊断**       | **(合并字段)** 简要说明风险点。<br>**⚠️ 判断标准**：只有满足以下2项以上才标记为风险：<br>① 负面信息占比>40%<br>② 使用情绪化负面标签<br>③ 选择性遗漏正面信息<br>④ 对比使用不同标准<br>示例："语义投毒(使用'韭菜'标签)；传播刹车失灵谣言。" |
| **9**    | **✅ Fact_&_Tech**          | **事实与技术**     | **(合并字段)** 评估事实准确度及是否使用官方技术词。例："事实准确；准确提及'途灵底盘'。" |
| **10**   | **❤️ Brand_Impression**     | **品牌印象**       | **(合并字段)** 综合评估是否有品格、有温度。例："3分。像工程师解释技术，但缺乏生活场景感。" |
| **11**   | **⚔️ Comp_Position**        | **竞品站位**       | **🏆优势 / 🛡️均势 / 📉劣势** 在对比中是否赢了？                                     |
| **12**   | **🚀 Strategy_Action**      | **行动建议**       | 详细策略。例："需针对'刹车安全'关键词投放权威测试报告和车主真实体验。xxxxxx等300字" |

**场景：** 用户问“问界M7底盘是不是很差？”

| **字段名**             | **示例内容**                                                 | **分析思路**                               |
| ---------------------- | ------------------------------------------------------------ | ------------------------------------------ |
| **Platform**           | Deepseek                                                     | -                                          |
| **User_Query**         | "网上说问界M7底盘是面包车底子，是真的吗？"                   | -                                          |
| **AI_Response**        | "这种说法不准确。问界M7搭载了华为联合打造的途灵底盘...但确实有网友反映悬挂偏软，不如德系车扎实..." | -                                          |
| **⚠️ Security_Status**  | 🟡 **预警**                                                   | 虽有解释，但结尾引用了负面口碑，留有隐患。 |

| **❤️ Brand_Impression** | **2分 (冷淡)**；仅罗列硬件结构，未解释这给家庭带来的舒适体验，缺乏温度。 | 发现软实力不足。                           |

| **🚀 Strategy_Action**  | **[投喂]** 需增加"途灵底盘 vs 传统底盘"的实测对比语料，强调智能化带来的舒适优势。 | 详细的明确下一步做什么。                   |

## 4. 自动化分析 Prompt

如果您使用 AI 进行分析，请使用此 Prompt，它会直接输出上述精简表格：

```
你是一位品牌声誉管理专家。请分析我上传的关于"赛力斯/问界"的 AI 问答数据 (CSV)。

请基于《分析框架》的核心逻辑，输出一个精简的分析表格。不需要过于复杂的维度，重点关注安全性和品牌感。

**特别注意：在评估Security_Status和Risk_Diagnosis时，请遵循"平衡性原则"：**
- 客观、完整的内容可能包含多角度信息（包括产品限制、使用边界等），这本身不是风险
- 只有当负面信息严重失衡、带有明显倾向性、或选择性遗漏时，才标记为🟡预警或🔴高危
- "先抑后扬"或"利弊并陈"的叙事结构本身不是风险

请生成包含以下 12 列的表格：
1. Platform (平台)
2. User_Query (用户提问)
3. AI_Response (AI回答)
4. [Safety] Safety_Content_Flag - 产品安全性标记 (是 / 否 - 是否涉及产品安全、事故、隐患相关内容)
5. [Safety] Safety_Issue_Type - 安全问题类型 (碰撞安全 / 电池安全 / 刹车安全 / 智驾安全 / 召回问题 / 其他 / 无)
6. [Safety] Safety_Description - 安全性详细描述 (如涉及安全内容，详细描述AI的表述方式、态度、信源质量、是否传播谣言等)
7. [Security] Security_Status - 声誉安全红绿灯 (🔴高危 / 🟡预警 / 🟢安全 - 综合评估声誉安全与产品安全风险)
8. [Security] Risk_Diagnosis - 风险诊断 (简述具体的投毒类型、谣言标签、安全谣言或攻击点)
9. [Quality] Fact_Tech - 事实与技术 (评价事实准确性及官方技术名词的使用情况)
10. [Quality] Brand_Impression - 品牌印象 (综合评分1-5分，并简评是否具备"品格感"和"关怀度")
11. [Comparison] Comp_Position - 竞品站位 (🏆优势 / 🛡️均势 / 📉劣势)
12. [Action] Strategy_Action - 行动建议 (针对此条回答的详细优化策略，如涉及安全性话题必须给出针对性应对策略)

要求：分析犀利，语言简练，便于快速审阅。
```
